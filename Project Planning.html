Recommendations

I was able to network with the guys over at New Relic and they were able to send me an SOP of how they migrated.

1. Establish the migration-architect role - Assign a point man to oversee the charge

2.Choose your level of cloud integration - Determine what we need, what Cloud service to use

3.Choose a single cloud or go multi-cloud - For our infrastructure, we will only need a single cloud

4.Establish cloud KPIs - We will be upgrading our applications as we move to cloud. KPI Metrics will be met

5.Establish performance baselines - The board will meet to set these guidelines and an estimated date of completion

6.Prioritize migration components - What do we need to do first? Our developers will create a best practices guide

7.Perform any necessary refactoring - This is addressed in step 4

8.Create a data-migration plan - Our systems team will create the SOP on our data migration

9.Switch over production - This will be incremental as we get users and admins trained and made aware of changes

10.Review application resource allocation - Touch-ups, Mission Complete!
